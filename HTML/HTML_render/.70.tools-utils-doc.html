Installation des outils

La bibliothèque Utils-latex regroupe les outils pour installer et compiler un répertoire de travail Latex.

Remarque

L'ensemble des outils présentés est optimisé pour une utilisation avec le logiciel Visual Studio Code
Il faut donc saisir la commande suivante dans le dossier où l'on souhaite créer le répertoire de travail : 

    git clone https://github.com/nicolasleguerroue/Utils-latex.git && Utils-latex/tools/init
On vous demande ensuite si vous souhaitez mettre à jour le système. Répondre y est fortement conseillé. A ce moment là, l'ensemble des logiciels et paquets nécessaires vont être installés. Enfin, on vous demande si vous souhaitez installer les extensions VScode, encore une fois, l'acceptation est vivement recommandée. Le logiciel VSCode se lance une fois que l'installation est terminée.

Organisation du projet

\createNoCenteredFigure{Arborescence du projet}{ \dirtree{% .1 Projet. .2 Images. .3 Intro. .3 Content. .3 OtherDirectory. .2 Make. .3 Alias.tex. .3 Bibliography.bib. .3 Contacts.tex. .3 Glossaries.tex. .3 Index.tex. .3 Nomenclature.tex. .3 Rules.tex. .3 Versions.tex. .2 Output. .2 Parts. .3 0.Intro. .3 1.Content. .3 2.OtherDirectory. .2 \sym{Utils}. .2 Utils-latex. .3 bash. .3 HTML. .3 tools. .3 Utils. .3 .vscode. .3 make. .2 \sym{.vscode}. .2 main.tex. .2 \sym{make}. } } Chaque projet est constitué de 8 dossiers et de 2 fichiers situés à la racine du projet. Les fichiers ou dossier violets (SYM) sont des liens symboliques dont la source est située dans le dossier Utils-latex.
  • Le dossier Images contient l'ensemble des images du projet.
    • Chaque image doit faire partie de la même partie que son document source associé. La gestion des emplacements des images est indiqué dans la section handleImages
  • Le dossier Make contient les fichiers annexes du projet:
    • Le fichier Alias.tex regroupe les les nouvelles commandes utilisés pour les abréviations.
    • Le fichier Bibliography.bib recense les bibliographies du projet. Pour plus d'informations, consulter la section biblio.
    • Le fichier Contacts.tex est une page pour contacter l'auteur et contient les informations sur les
      • droits et les licences du projet.
    • Le fichier Glossaries.tex contient le glossaire. Pour ajouter une définition,
      • veuillez vous référer à la section addDef
    • Le fichier Index.tex contient l'index.
    • Le fichier Nomenclature.tex contient la structure de la nomenclature\footnote{Les unités et
      • grandeurs physiques par exemple}. Pour ajouter un element, consulter la section addNomenclature
    • Le fichier Rules.tex contient les conventions pour le projet. Il peut contenir les types de commandes,
      • les conventions de nommage du projet...
    • Le fichier Versions.tex contient les différentes versions du projet. Pour plus d'information,
      • consulter la partie addVersion
  • Le dossier Output contient les fichiers de compilation générés de manière automatique.
    • Vous n'aurez pas à modifier des fichiers à cette emplacement.
  • Le dossier Parts contient les différentes parties du projet. Il est possible de scinder son projet
    • en grandes parties (Introduction, Chapitre1, Chapitre2, Conclusion), chaque dossier contenu dans le dossier Parts représente ces parties. Dans chacun de ces dossiers, vous pouvez créer autant de fichier Latex que vous voulez, il seront compilés dans l'ordre alphabétique ou bien par ordre croissant si vous mettre un numéro au début du nom de fichier. Pour nommer les dossiers dans le dossier Parts, \colors{red}{il faut impérativement commencer le nom avec un numéro suivi d'un point.} Par exemple 0.Intro puis 1.Content. Les espaces sont interdits dans les noms des dossiers mais les symboles ~ ~ et - sont acceptés. Pour chaque dossier crée dans le dossier Parts, il faudra créer un dossier avec le même nom dans le dossier Images, sous peine de voir une volée d'erreurs lors de la compilation.
  • Le dossier Settings qui contient les fichiers de configuration du projet.
    • Pour configurer le projet, reportez-vous à la section setSettings :
    • Le fichier ChapterAlias.tex enregistre le nom des chapitres ('Section' au lieu de 'Chapitre' par exemple)
    • Le fichier DocumentClass.tex sauvegarde la classe du document.
    • Le fichier FHeaderLine.tex s'occupe de l'épaisseur des lignes entre l'en-tête et le pied de page.
    • Le fichier Fonts.tex sélectionne la police du document.
    • Le fichier Hyperref.tex enregistre les métadonnées du fichier.
    • Le fichier Includes.tex est un fichier généré automatiquement pour importer tous les paramètres cités.
    • Le fichier Margins.tex s'occupe des marges du document.
    • Le fichier NomenclatureName.tex enregistre le nom de la nomenclature.
    • Le fichier Theme.tex enregistre le thème des pages.
    • Le fichier TocDepth.tex enregistre la profondeur du sommaire.
  • Le dossier \sym{Utils} est un dossier symbolique qui contient les bibliothèques Latex du projet. Le fichier Utils.sty est généré
    • dynamiquement, toute écriture manuelle sera écrasée à la prochaine compilation.
  • Le dossier Utils-latex contient toutes les bibliothèques du projet ainsi que les outils de compilation.
    • Il est configuré en tant que sous-module Git afin de séparer le contenu des bibliothèques et le contenu Latex à compiler. Ce dossier contient :
    • Le dossier bash qui regroupe plusieurs bibliothèques :
      • La bibliothèque Charts.sh qui permet de générer des graphiques
      • La bibliothèque Colors.sh qui gère les couleurs dans le terminal
      • La bibliothèque Directories.sh qui permet de gérer les fichiers et dossiers
      • La bibliothèque Networks.sh qui surveille l'état de la connexion Internet
      • La bibliothèque System.sh qui permet de faire des fonctions System
      • La bibliothèque Time.sh qui gère les dates
    • Le dossier HTML qui regroupe plusieurs bibliothèques utilisées pour convertir le code source Latex en fichier HTML.:
      • La bibliothèque Badge.py qui gère les badges
      • La bibliothèque Color.py qui gère les couleurs
      • La bibliothèque Command.py qui gère les types de commandes disponibles
      • La bibliothèque Glossary.py qui gère le glossaire
      • La bibliothèque Image.py qui gère les images et leur intégration au sein du fichier HTML[note]
      • La bibliothèque Label.py qui gère labels
      • Le fichier generateHTML.py qui génère les pages HTML
      • Le fichier startWebServer.py qui démarre un serveur Web pour afficher la page Web de rendu
      • Le dossier HTML render contient les pages Web générées
    • Le dossier tools regroupe plusieurs fichiers :
      • Le fichier checking qui permet de vérifier l'intégrité du dossier de projet lors du lancement de VScode
      • Le fichier init qui initialise le dossier du projet
      • Le fichier install qui installe les paquets et utilitaires pour la compilation
      • Le fichier compileAllFiles.py désactive la compilation pour l'ensemble des dossiers contenus
        • dans le dossier Parts. \footnote{Concrètement, cela revient à ajouter un point devant le nom du dossier pour que le compilateur ignore le dossier.}
      • Le fichier compileAnyFile.py autorise la compilation des dossiers contenus dans le dossier
        • Parts. [note]
  • Le dossier \sym{.vscode} contient tous les fichiers de configuration du répertoire VScode :
    • Le fichier generateSnippets.php créer un fichier de snippets VScode
      • afin de gérer l'autocomplétion VSCode. \bold{Ce fichier doit obligatoirement être appelé depuis le fichier make.}
    • Le fichier settings.json contient les tâches exécutables VScode via un bouton [note]}}. Ces tâches permettent de lancer la compilation et les utilitaires du répertoire.
    • Le fichier task.json est le fichier qui recense les tâches exécutables sous VSCode.
    Et voici les deux fichiers situés à la racine:
    • Le fichier main.tex est le fichier principal du projet. C'est dans ce fichier qu'on définit notamment :
      • Le choix de la classe du document.
      • Le titre de la page
      • Le choix de la disposition du document
      • Le choix de la présence des fichiers dans le dossier Make, c'est à dire les fichiers
        • personnalisables tels que la bibliographie, l'index, le glossaire...
    • Le fichier \sym{make} est le fichier de compilation. \bold{Vous n'aurez pas besoin de modifier ce
      • fichier pour une utilisation classique}
La compilation du projet se fait grâce au fichier make situé à la racine du projet. L'ensemble des outils de compilation sont disponibles de deux façons :
  • Via un terminal
  • Via le logiciel VSCode
L'avantage du logiciel VScode est qu'il fournit des raccourcis clavier et une interface graphique plus évoluée (boutons)

Compilation avec un terminal

Compilation classique

Une compilation classique a pour objectif de générer le fichier PDF de rendu, appelé main.pdf et situé à la racine du projet. La commande est la suivante : 

./make --full
Lors de la compilation, plusieurs fichiers sont générés à la racine, dont :
  • Le fichier .render report.tex (fichier caché) qui contient la première partie des fichiers journaux de compilation
  • Le fichier .render report logs.tex qui contient la seconde partie des fichiers journaux de compilation[note]
  • Une image Part.png qui affiche le nombre de ligne pour chaque fichier compilé contenu dans le dossier Parts
  • Une image Utils.png qui affiche le nombre de ligne pour chaque fichier contenu dans le dossier Utils
  • Un fichier standlone.tex qui contient l'ensemble du code latex généré. Ne pas supprimer ce fichier, il est utilisé pour la conversion du code Latex en fichier source HTML.

Ajout de version

Il est possible d'ajouter une version de projet en invoquant la commande suivante : 

./make --version
La date de la mise à jour vous sera demandée ainsi que le contenu de la mise à jour.

Mise à jour de l'autocomplétion

Il est possible de mettre à jour l'autocomplétion sous VScode pour les bibliothèques Utils. En invoquant le paramètre --snippet, il est possible de générer le fichier VScode qui va ajouter l'autocomplétion. Ce fichier est appelée output.snippet-code et se situe dans le dossier .vscode 

./make --snippet

Compilation avec VSCode

L'ensemble des commandes présentées précédemment sont disponibles sous VSCode.

Ouverture d'un projet

Il faut ouvrir un terminal à la racine du projet et lancer la commande : 

code .

Remarque

Lors de chaque ouverture du répertoire de projet VSCode, un script de vérification des fichier va se lancer. Les fichiers manquants seront regénérés à leur valeur par défaut.
Le logiciel se lance avec une interface similaire : On observe sur la gauche l'arborescence du projet. En cliquant sur les dossiers, ces derniers se déroulent et affichent leur contenu.

Lancement de la compilation

Il y deux méthodes pour lancer les outils de compilation sous VSCode :
  • Via les raccourcis clavier (Tâches)
  • Via les boutons sur l'interface graphique

Les raccourcis clavier

Les commandes sont classées par catégorie, à savoir :
  • Compilation
    • Les commandes pour compiler le projet.
  • Settings
    • Les commandes pour définir les paramètres du projet.
  • Display
    • Les commandes pour afficher ou non des éléments (liste des figures...)
  • Part
    • Les commandes pour gérer les parties

Les commandes de compilation

  • Compilation
    • Full
      • Lance une compilation complète
    • Display HTML render
      • Lance la page d'affichage du rendu HTML du code Latex. Avant de faire cette commande, il faut faire la commande Compilation > Convert Latex to HTML
    • Enable all files
      • Autorise la compilation complète du répertoire Parts
    • Disable all files
      • Empêche la compilation complète du répertoire Parts
    • Display PDF
      • Ouvre le PDF de sortie (main.pdf) avec le logiciel okular
    • Display errors
      • Affiche le fichier de sortie de compilation.
    • Convert Latex to HTML
      • Lance les scripts python pour convertir le Latex en HTML.
    • Light (without bibliography, index and nomenclature)
      • Lance une compilation rapide sans bibliographie, index et nomenclature. Cette compilation ne compile pas l'intégralité du document, la table des matière sera donc absente mais cela a le mérite de compiler rapidement.
Pour trier les commandes disponibles, il faut saisir le type de commande (Compilation, Settings...) et les commandes seront triées.

Les commandes de configuration du projet

Il est possible d'éditer de nombreux paramètres de manière graphique sous VScode. Les configurations se font via les commandes de type Settings
  • Settings
    • Depth of TOC
      • Définit la profondeur du sommaire (défaut=1)
    • Update of font
      • Définit la police du document.
    • Update of theme
      • Définit le thème du document
    • Update of margins
      • Définit les marges du document.
    • Update of metadata
      • Ouvre le fichier de définition des métadonnées
    • Update of line widht
      • Définit l'épaisseur des traits pour les en-tête et pied de page.
    • Update of document class
      • Définit la classe du document
    • Update of nomenclature
      • Définit le nom de la nomenclature.

Les boutons

Au lieu d'utiliser les raccourcis avec CTRT+SHIFT+B, il est possible d’exécuter les outils avec les boutons en bas de VSCode : Le choix d'un dossier par partie (Parts/XXX) permet de fusionner très facilement des projets. Pour fusionner deux projets, il suffit de copier-coller le contenu du dossier Images et Parts du projet A dans le dossier de projet qui contiendra la fusion (projet B). Lors de la compilation, make va gérer la fusion automatiquement.
Header
veut dire que le code est à mettre avant begin\{document\}
Body
veut dire que le code est à mettre entre begin\{document\} et end\{document\}
main.tex
veut dire que le code est à mettre dans le fichier main.tex
Make/Nomenclature.tex
veut dire que le code est à mettre dans le fichier Nomenclature.text situé dans le dossier Make
Make/Bibliography.bib
veut dire que le code est à mettre dans le fichier Bibliography.bib situé dans le dossier Make

La bibliothèque Bibliography gère la bibliographie du document.

Format de la bibliographie

La bibliographie contenue dans le fichier Make/Bibliography.bib doit être au format Bibtex :
Make/Bibliography.bib

    @book{ID,

    title={Titre},

    author={Auteur},

    year={2007},

    note={Une description},

    publisher={Editeur}

},

 ...
Le format Zotero se base sur cette architecture.

Affichage de la bibliographie

Pour afficher la bibliographie située dans le dossier Make/Bibliography.bib, il faut saisir la commande suivante :
main.tex

\displayBibliography{Bibliographie}{Make/Bibliography}

Remarque

L'extension du fichier .bib ne doit pas être précisée !

La Bibliothèque Nomenclature permet de générer des nomenclatures.

Ajout d'un élément dans la nomenclature

Chaque élément est de la forme 

\nomenclature[category]{\(symbole\)}{définition}

%Par exemple

\nomenclatu\(r\)e[E]{\(r\)}{Rappo\(r\)t cyclique d'un signal pé\(r\)iodique}
et doit être rajouté dans le fichier Make/Nomenclature.tex Dans la commande, category indique le type de grandeur. Il y a actuellement 6 types de grandeurs définies dans le fichier Utils/Nomenclature.sty[note]:
  • P pour les Constantes physiques
  • O pour les Autres symboles
  • N pour les Nombres spéciaux
  • A pour les Amplificateurs Opérationnels
  • M pour les Mécanique
  • E pour les Électronique

Ajout des unités

Pour ajouter une unité, il suffit d'invoquer la commande addUnit à la fin du dernier argument de la commande nomenclature.

Ajout d'une unité


Make/Nomenclature.tex

\nomenclature[A]{\(\varepsilon\)}{Tension différentielle (\(\varepsilon\) = E_+ - E_-)\addUnit{V}}

Exemple de nomenclature

La nomenclature est dans le fichier Make/Nomenclature.tex.
Make/Nomenclature.tex

\nomenclatu\(r\)e[E]{\(r\)}{Rappo\(r\)t cyclique d'un signal pé\(r\)iodique}

\nomenclature[A]{\(E_+\)}{Tension entrée non inverseuse \addUnit{V}}

\nomenclature[A]{\(E_-\)}{Tension entrée inverseuse \addUnit{V}}

\nomenclature[E]{\(\eta\)}{Rendement d'un mécanisme \addUnit{\%}}

\nomenclature[E]{\(\varphi\)}{Déphasage entre deux signaux \addUnit{rad}}
