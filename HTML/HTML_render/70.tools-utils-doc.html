Introduction

Présentation

Ce document a pour but de présenter les fonctionnalités de la bibliothèque Utils, qui n'est qu'un regroupement de bibliothèques pour simplifier l'utilisation de Latex. Installation des outils

La bibliothèque Utils-latex regroupe les outils pour installer et compiler un répertoire de travail Latex.

\faviconInfo

L'ensemble des outils présentés est optimisé pour une utilisation avec le logiciel Visual Studio Code
Il faut donc saisir la commande suivante dans le dossier où l'on souhaite créer le répertoire de travail : 

git clone https://github.com/nicolasleguerroue/Utils-latex.git && Utils-latex/tools/init
On vous demande ensuite si vous souhaitez mettre à jour le système. Répondre y est fortement conseillé. A ce moment là, l'ensemble des logiciels et paquets nécessaires vont être installés. Voici les paquets installés :
  • texlive-full
  • git
  • texlive-lang-european
  • okular
  • gnuplot
Enfin, on vous demande si vous souhaitez installer les extensions VScode, encore une fois, l'acceptation est vivement recommandée. Le logiciel VSCode se lance une fois que l'installation est terminée.

Configuration des tâches

Lors de l'ouverture du répertoire de compilation, un script de vérification va se lancer automatiquement. Pour ce faire, il va falloir activer le lancement des tâches. Il faut faire le raccourci clavier CTRL+SHIFT+P et saisir Manage Automatic tasks in Folder Puis valider l'action avec Manage Automatic tasks in Folder Ce script exécuté au démarrage permet de vérifier l'intégrité des fichiers (paramètres, outils) avant de lancer une compilation.

Organisation du projet

\createNoCenteredFigure{Arborescence du projet}{ \dirtree{% .1 Projet. .2 Images. .3 Intro. .3 Content. .3 OtherDirectory. .2 Make. .3 Alias.tex. .3 Bibliography.bib. .3 Colors.tex. .3 Contacts.tex. .3 Glossaries.tex. .3 Index.tex. .3 Nomenclature.tex. .3 Rules.tex. .3 Versions.tex. .2 Output. .2 Parts. .3 0.Intro. .3 1.Content. .3 2.OtherDirectory. .2 Settings. .2 Utils. .2 Utils-latex. .3 bash. .3 HTML. .3 tools. .3 Utils. .3 .vscode. .3 make. .2 .vscode. .2 main.tex. .2 make. } } Chaque projet est constitué de 8 dossiers et de 2 fichiers situés à la racine du projet. Les fichiers ou dossiers violets (SYM) sont des liens symboliques dont la source est située dans le dossier Utils-latex.
  • Le dossier Images contient l'ensemble des images du projet.
    • Chaque image doit faire partie de la même partie que son document source associé. La gestion des emplacements des images est indiqué dans la section handleImages
  • Le dossier Make contient les fichiers annexes du projet:
    • \begin{items}{white}{}
  • Le fichier Alias.tex regroupe les nouvelles commandes utilisées pour les abréviations.
  • Le fichier Bibliography.bib recense la bibliographie du projet. Pour plus d'informations, consulter la section biblio.
  • Le fichier Colors.tex est un fichier pour centraliser les nouvelles couleurs définies.
    • Pour ajouter une couleur, veuillez vous référer à la section colors
  • Le fichier Contacts.tex est une page pour contacter l'auteur et contient les informations sur les
    • droits et les licences du projet.
  • Le fichier Glossaries.tex contient le glossaire. Pour ajouter une définition,
    • veuillez vous référer à la section addDef
  • Le fichier Index.tex contient l'index. Pour plus d'informations, consulter la section index.
  • Le fichier Nomenclature.tex contient la structure de la nomenclature\footnote{Les unités et
    • grandeurs physiques par exemple}. Pour ajouter un élément, consulter la section addNomenclature
  • Le fichier Rules.tex contient les conventions pour le projet. Il peut contenir les types de commandes,
    • les conventions de nommage du projet \ldots
  • Le fichier Versions.tex contient les différentes versions du projet. Pour plus d'information,
    • consulter la partie addVersion
  • Le dossier Output contient les fichiers de compilation générés de manière automatique.
    • Vous n'aurez pas à modifier les fichiers à cette emplacement.
  • Le dossier Parts contient les différentes parties du projet. Il est possible de scinder son projet
    • en grandes parties (Introduction, Chapitre1, Chapitre2, Conclusion), chaque dossier contenu dans le dossier Parts représente ces parties. Dans chacun de ces dossiers, vous pouvez créer autant de fichiers Latex que vous voulez, il seront compilés dans l'ordre croissant si vous mettre un numéro au début du nom de fichier. Pour nommer les dossiers dans le dossier Parts, \colors{red}{il faut impérativement commencer le nom avec un numéro suivi d'un point.} Par exemple 0.Intro puis 1.Content. Les espaces sont interdits dans les noms des dossiers mais les symboles ~ ~ et - sont acceptés. Pour chaque dossier crée dans le dossier Parts, il faudra créer un dossier avec le même nom (sans le numéro et le point) dans le dossier Images, sous peine de voir une volée d'erreurs lors de la compilation.
  • Le dossier Settings qui contient les fichiers de configuration du projet.
    • Pour configurer le projet, reportez-vous à la section setSettings \begin{items}{white}{}
  • Le fichier ChapterAlias.tex enregistre le nom des chapitres ('Section' au lieu de 'Chapitre' par exemple).
  • Le fichier DocumentClass.tex sauvegarde la classe du document.
  • Le fichier FHeaderLine.tex s'occupe de l'épaisseur des lignes entre l'en-tête et le pied de page.
  • Le fichier Fonts.tex sélectionne la police du document.
  • Le fichier Header.tex affiche la première page
  • Le fichier Hyperref.tex enregistre les métadonnées du fichier.
  • Le fichier Includes.tex est un fichier généré automatiquement pour importer tous les paramètres cités.
  • Le fichier ListMargin.tex règle l'indentation des listes.
  • Le fichier LOF.tex autorise l'affiche de la liste des figures
  • Le fichier Margins.tex s'occupe des marges du document.
  • Le fichier NomenclatureName.tex enregistre le nom de la nomenclature.
  • Le fichier Theme.tex enregistre le thème des pages.
  • Le fichier TocDepth.tex enregistre la profondeur du sommaire.
  • Le fichier TocSize.tex enregistre l'agencement du sommaire.
  • Le fichier SectionColor.tex enregistre la couleur des sections.
  • Le fichier SubSectionColor.tex enregistre la couleur des sous-sections.
  • Le fichier TitlePrefix.tex enregistre le préfixe des sections et sous-sections.
  • Le dossier Utils est un dossier symbolique qui contient les bibliothèques Latex du projet. Le fichier Utils.sty est généré
    • dynamiquement, toute écriture manuelle sera écrasée à la prochaine compilation.
  • Le dossier Utils-latex contient toutes les bibliothèques du projet ainsi que les outils de compilation.
    • Il est configuré en tant que sous-module Git afin de séparer le contenu des bibliothèques et le contenu Latex à compiler. Ce dossier contient :

    • \begin{items}{blue}{}

  • Le fichier generateHTML.py qui génère les pages HTML
  • Le fichier startWebServer.py qui démarre un serveur Web pour afficher la page Web de rendu
  • Le dossier HTML render contient les pages Web générées
  • Le dossier tools regroupe plusieurs fichiers :
    • \begin{items}{blue}{}

  • Le fichier init qui initialise le dossier du projet
  • Le fichier install qui installe les paquets et utilitaires pour la compilation
  • Le fichier compileAnyFiles.py désactive la compilation pour l'ensemble des dossiers contenus
    • dans le dossier Parts. \footnote{Concrètement, cela revient à ajouter un point devant le nom du dossier pour que le compilateur ignore le dossier.}

    • \begin{items}{cyan}{}

  • Le fichier task.json est le fichier qui recense les tâches exécutables sous VSCode.
    • Et voici les deux fichiers situés à la racine:
    • Le fichier main.tex est le fichier principal du projet. C'est dans ce fichier qu'on définit notamment :
      • L'import de la classe du document
      • Le titre de la page
      • Le choix de la disposition du document
      • Le choix de la présence des fichiers dans le dossier Make, c'est à dire les fichiers
        • personnalisables tels que la bibliographie, l'index, le glossaire\ldots
    • Le fichier make est le fichier de compilation. \bold{Vous n'aurez pas besoin de modifier ce
      • fichier pour une utilisation classique}
    La compilation du projet se fait grâce au fichier make situé à la racine du projet. L'ensemble des outils de compilation sont disponibles de deux façons :
    • Via un terminal
    • Via le logiciel VSCode
    L'avantage du logiciel VScode est qu'il fournit des raccourcis clavier et une interface graphique plus évoluée (boutons)

    Compilation avec un terminal

    Compilation classique

    Une compilation classique a pour objectif de générer le fichier PDF de rendu, appelé main.pdf et situé à la racine du projet. La commande est la suivante : 
    
./make --full
    Lors de la compilation, plusieurs fichiers sont générés à la racine, dont :
    • Le fichier .render report.tex (fichier caché) qui contient la première partie des fichiers journaux de compilation
    • Le fichier .render report logs.tex qui contient la seconde partie des fichiers journaux de compilation[Note 13]
    • Une image Part.png qui affiche le nombre de ligne pour chaque fichier compilé contenu dans le dossier Parts
    • Une image Utils.png qui affiche le nombre de ligne pour chaque fichier contenu dans le dossier Utils
    • Un fichier standlone.tex qui contient l'ensemble du code latex généré. Ne pas supprimer ce fichier, il est utilisé pour la conversion du code Latex en fichier source HTML.
    Lors de la compilation, différents messages s'affichent :

    Ajout de version

    Il est possible d'ajouter une version de projet en invoquant la commande suivante : 
    
./make --version
    La date de la mise à jour vous sera demandée ainsi que le contenu de la mise à jour.

    Mise à jour de l'autocomplétion

    Il est possible de mettre à jour l'autocomplétion sous VScode pour les bibliothèques Utils. En invoquant le paramètre --snippet, il est possible de générer le fichier VScode qui va ajouter l'autocomplétion. Ce fichier est appelée output.snippet-code et se situe dans le dossier .vscode 
    
./make --snippet
    La commande se termine en affichant le nombre de commandes documentées.

    Compilation avec VSCode

    L'ensemble des commandes présentées précédemment sont disponibles sous VSCode.

    Ouverture d'un projet

    Il faut ouvrir un terminal à la racine du projet et lancer la commande : 
    
code .

    Lors de chaque ouverture du répertoire de projet VSCode, un script de vérification des fichiers va se lancer. Les fichiers manquants seront regénérés à leur valeur par défaut.
    Le logiciel se lance avec une interface similaire : On observe sur la gauche l'arborescence du projet. En cliquant sur les dossiers, ces derniers se déroulent et affichent leur contenu.

    Lancement de la compilation

    Il y deux méthodes pour lancer les outils de compilation sous VSCode :
    • Via les raccourcis clavier (Tâches)
    • Via les boutons sur l'interface graphique

    Les raccourcis clavier

    Les commandes sont classées par catégorie, à savoir :
    • Compilation
      • Les commandes pour compiler le projet.
    • Settings
      • Les commandes pour définir les paramètres du projet.
    • Display
      • Les commandes pour afficher ou non des éléments (liste des figures...)
    • Part
      • Les commandes pour gérer les parties

    Les commandes de compilation

    Les différentes commandes sont de 3 types :
    • Une commande qui lance un script en arrière-plan, sans argument, repérée avec la mention white{darkOrange}{script}
    • Une liste avec valeurs déterminées, repérées avec la mention white{cyan}{élement}.La valeur par défaut est de la couleur bleue foncée (white{blue}{Defaut})
    • Zone de saisie utilisateur, repérée avec la mention white{purple}{Entrée utilisateur}
    • Compilation
      • Full
        • Lance une compilation complète white{darkOrange}{script}
      • Display HTML render
        • Lance la page d'affichage du rendu HTML du code Latex. Avant de faire cette commande, il faut faire la commande Compilation > Convert Latex to HTML white{darkOrange}{script}
      • Enable all files
        • Autorise la compilation complète du répertoire Parts white{darkOrange}{script}
      • Disable all files
        • Empêche la compilation complète du répertoire Parts white{darkOrange}{script}
      • Display PDF
        • Ouvre le PDF de sortie (main.pdf) avec le logiciel okular white{darkOrange}{script}
      • Display errors
        • Affiche le fichier de sortie de compilation white{darkOrange}{script}
      • Convert Latex to HTML
        • Lance les scripts python pour convertir le Latex en HTML white{darkOrange}{script}
      • Light (without bibliography, index and nomenclature)
        • Lance une compilation rapide sans bibliographie, index et nomenclature. Cette compilation ne compile pas l'intégralité du document, la table des matière sera donc absente mais cela a le mérite de compiler rapidement. white{darkOrange}{script}
    Pour trier les commandes disponibles, il faut saisir le type de commande (Compilation, Settings...) et les commandes seront triées.

    Autocomplétion des commandes

    Lors du démarrage de l'espace de travail, un script va générer le fichier pour l'autocomplétion des commandes. Chaque commande commence par le mot clé "Utils" puis un point et le nom de la bibliothèque associée. Par exemple, pour obtenir les commandes de la bibliothèque Images, en écrivant "Images" dans un document Latex vous obtiendrez le résultat suivant : Vous pouvez parcourir les commandes disponibles puis saisir la touche Enter. La commande sera écrite à la place du curseur. Le choix d'un dossier par partie (Parts/XXX) permet de fusionner très facilement des projets. Pour fusionner deux projets, il suffit de copier-coller le contenu du dossier Images et Parts du projet A dans le dossier de projet qui contiendra la fusion (projet B). Lors de la compilation, make va gérer la fusion automatiquement.

    Format de la bibliographie

    La bibliographie contenue dans le fichier Make/Bibliography.bib doit être au format Bibtex :
    Make/Bibliography.bib
    
    @book{ID,

    title={Titre},

    author={Auteur},

    year={2007},

    note={Une description},

    publisher={Editeur}

},

 ...
    Le mot-clé ID permettra par la suite de référencer l'élément dans la bibliographie. Le format Zotero se base sur cette architecture.

    Affichage de la bibliographie

    Pour afficher la bibliographie située dans le dossier Make/Bibliography.bib, il faut saisir la commande suivante :
    main.tex
